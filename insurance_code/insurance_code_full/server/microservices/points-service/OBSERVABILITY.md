# Points Service Observability

Week7 C-side observability baseline for `points-service`.

## Runtime metrics inventory

`points-service` now keeps an internal runtime snapshot for the core transaction chain:

| Metric | Meaning | Success path | Failure path |
| --- | --- | --- | --- |
| `activityReward.successRate` | Activity completion reward settlement success rate | `settleActivityReward()` completes and posts to points | Validation or settlement failure before points posting finishes |
| `learningReward.successRate` | Learning completion reward settlement success rate | `settleLearningCourseReward()` completes and posts to points | Validation or settlement failure before points posting finishes |
| `signIn.successRate` | Daily sign-in success rate | `POST /api/sign-in` returns `200` | Any failed sign-in request, including `401`, `403`, `409`, `400` |
| `redeem.successRate` | Mall redeem success rate | `POST /api/mall/redeem` returns `200` | Any failed redeem request, including auth/csrf/stock/points errors |
| `writeoff.successRate` | Redemption writeoff success rate | `POST /api/redemptions/:id/writeoff` returns `200` | Any failed writeoff request, including auth/csrf/token/idempotency errors |
| `orderTransitions` | Order status transition counts | `none -> created`, `created -> paid`, `paid -> fulfilled`, `paid -> cancelled`, `created -> cancelled` | No transition is recorded for idempotent no-op requests |
| `pointsMovements.creditCount` | Number of successful points increases | Sign-in reward, mall activity reward, refund/cancel refund | Not incremented for duplicated idempotent writes |
| `pointsMovements.debitCount` | Number of successful points decreases | Order pay / mall redeem | Not incremented for duplicated idempotent writes |
| `errorCounts` | Error code frequency for points routes | Captured from route catch blocks and middleware JSON errors | Includes auth/csrf/validation failures before route code executes |

Internal runtime snapshot endpoint:

- `GET /internal/points-service/observability`

Response sections:

- `metrics`
- `logFieldSpec`
- `recentLogs`
- `pendingBridgeMethods`
- `errorCatalog`

## Log field spec

All points transaction-chain request logs now carry these normalized fields:

| Field | Required | Meaning |
| --- | --- | --- |
| `trace_id` | Yes | Reuses inbound `x-trace-id`; otherwise generated by `points-service` |
| `user_id` | No | Current customer id resolved from bearer session |
| `order_id` | No | Related order id for redeem/order/writeoff flows |
| `redemption_id` | No | Related redemption id for redeem/writeoff flows |
| `route` | Yes | Request route in `METHOD /path` format |
| `result` | Yes | Normalized result: `success` or `fail` |

Current structured request log also includes supporting fields:

- `request_id`
- `status_code`
- `duration_ms`
- `error_code`
- `error_category`
- `ts`

Output location:

- standard output as JSON log lines prefixed with `[points-service]`
- in-memory rolling window under `recentLogs` from `GET /internal/points-service/observability`
- internal cross-domain settlements also emit synthetic logs into `recentLogs`
- Week15 activity reward route name:
  - `INTERNAL activity->points reward settlement`
- Week13 learning reward route name:
  - `INTERNAL learning->points reward settlement`

## Transaction error classification

| Code | Category | Primary routes | Meaning |
| --- | --- | --- | --- |
| `INVALID_ACTIVITY_REWARD_USER` | `validation` | `INTERNAL activity->points reward settlement` | Missing or invalid activity reward customer id |
| `INVALID_ACTIVITY_REWARD_ACTIVITY_ID` | `validation` | `INTERNAL activity->points reward settlement` | Missing or invalid activity id |
| `INVALID_ACTIVITY_REWARD_DATE` | `validation` | `INTERNAL activity->points reward settlement` | Activity reward completion date is invalid |
| `INVALID_ACTIVITY_REWARD_POINTS` | `business_rule` | `INTERNAL activity->points reward settlement` | Activity reward points are invalid |
| `ACTIVITY_REWARD_SETTLEMENT_FAILED` | `system` | `INTERNAL activity->points reward settlement` | Activity reward settlement failed before posting completed |
| `INVALID_LEARNING_REWARD_USER` | `validation` | `INTERNAL learning->points reward settlement` | Missing or invalid learning reward customer id |
| `INVALID_LEARNING_REWARD_COURSE_ID` | `validation` | `INTERNAL learning->points reward settlement` | Missing or invalid course id |
| `INVALID_LEARNING_REWARD_POINTS` | `business_rule` | `INTERNAL learning->points reward settlement` | Learning course reward points are invalid |
| `LEARNING_REWARD_SETTLEMENT_FAILED` | `system` | `INTERNAL learning->points reward settlement` | Learning reward settlement failed before posting completed |
| `NEED_BASIC_VERIFY` | `auth_precondition` | `POST /api/sign-in`, `POST /api/mall/redeem` | User basic verification missing |
| `UNAUTHORIZED` | `auth` | `POST /api/sign-in`, `POST /api/redemptions/:id/writeoff` | Bearer token missing or invalid |
| `ALREADY_SIGNED` | `idempotency` | `POST /api/sign-in` | Duplicate same-day sign-in |
| `INVALID_ITEM_ID` | `validation` | `POST /api/mall/redeem` | Invalid item id |
| `ITEM_NOT_FOUND` | `resource` | `POST /api/mall/redeem`, `POST /api/orders`, `POST /api/orders/:id/pay` | Product not found |
| `ITEM_NOT_AVAILABLE` | `business_rule` | `POST /api/mall/redeem` | Product not redeemable |
| `OUT_OF_STOCK` | `capacity` | `POST /api/mall/redeem`, `POST /api/orders`, `POST /api/orders/:id/pay` | Stock insufficient |
| `INSUFFICIENT_POINTS` | `balance` | `POST /api/mall/redeem`, `POST /api/orders/:id/pay` | Balance insufficient |
| `INVALID_ACTIVITY_ID` | `validation` | `POST /api/mall/activities/:id/join` | Invalid activity id |
| `MALL_ACTIVITY_NOT_FOUND` | `resource` | `POST /api/mall/activities/:id/join` | Activity not found |
| `MALL_ACTIVITY_NOT_AVAILABLE` | `business_rule` | `POST /api/mall/activities/:id/join` | Activity unavailable |
| `NO_PERMISSION` | `authorization` | `GET /api/mall/items`, `GET /api/mall/activities`, `POST /api/mall/activities/:id/join` | Tenant/actor access denied |
| `INVALID_PRODUCT_ID` | `validation` | `POST /api/orders` | Invalid product id |
| `INVALID_ORDER_ID` | `validation` | `GET /api/orders/:id`, order pay/cancel/refund | Invalid order id |
| `ORDER_NOT_FOUND` | `resource` | `GET /api/orders/:id`, order pay/cancel/refund | Order not found |
| `ORDER_FORBIDDEN` | `authorization` | `GET /api/orders/:id`, order pay/cancel | Order belongs to another user |
| `ORDER_ALREADY_FULFILLED` | `business_rule` | order cancel/refund | Order already shipped or written off |
| `ORDER_NOT_PAID` | `business_rule` | order refund, redemption writeoff | Unpaid order cannot continue |
| `REDEMPTION_NOT_FOUND` | `resource` | `POST /api/redemptions/:id/writeoff` | Redemption missing |
| `ALREADY_WRITTEN_OFF` | `idempotency` | `POST /api/redemptions/:id/writeoff` | Duplicate writeoff |
| `INVALID_TOKEN` | `validation` | `POST /api/redemptions/:id/writeoff` | Writeoff token mismatch |
| `TOKEN_EXPIRED` | `business_rule` | `POST /api/redemptions/:id/writeoff` | Writeoff token expired |
