# 错误码字典（v1）

更新时间：2026-03-06

## 1. 目的

- 统一前端与后端错误码口径，避免“接口加了错误码但文档未更新”。
- 作为 API 合约的一部分，进入 CI 校验。

## 2. 状态码口径

- `400`：参数非法、上下文缺失、业务输入不满足前置条件。
- `401`：未登录或登录态失效（前端需触发登出清理）。
- `403`：有登录态但无权限、缺少实名、CSRF 不通过。
- `404`：目标资源不存在。
- `409`：业务冲突（库存不足、积分不足、重复操作、状态冲突）。
- `410`：令牌或资源过期。
- `413`：请求体过大。
- `429`：频控限制。
- `500`：系统内部错误。

## 3. 前端处理口径

- 触发强制清理会话的错误：`UNAUTHORIZED`、`CSRF_INVALID`。
- 业务弹窗/提示类错误：例如 `NEED_BASIC_VERIFY`、`INSUFFICIENT_POINTS`、`OUT_OF_STOCK`。
- 所有新错误码必须在本文件登记，并通过 `npm run docs:check:error-codes`。
