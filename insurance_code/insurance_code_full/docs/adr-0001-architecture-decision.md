# ADR-0001 架构执行线收敛

更新时间：2026-03-01  
状态：Accepted（当前唯一执行基线）

## 背景

当前仓库同时存在两条架构路线文档：

- Node 模块化单体（短期可执行）
- Java 三服务迁移（中长期规划）

并行推进会导致团队执行分叉、评审标准不一致、联调节奏失控。

## 决策

在 2026-Q2 之前，**唯一执行线**为：

1. 以 Node + Express 现有后端为生产改造基线。
2. 按“模块化单体”完成安全、权限、租户隔离、事件指标标签链路治理。
3. Java 三服务仅保留为“迁移设计输入”，不作为当前迭代的交付目标。

## 落地边界

### In Scope（当前迭代必须执行）

- 修复默认兜底（tenant/org/team/actor）路径。
- 强制登录与上下文来源统一（禁止伪造请求头覆盖登录态）。
- 路由拆分与服务层抽离（降低超大路由文件耦合）。
- 契约冻结、回归自动化、浏览器回归基线执行。

### Out of Scope（当前迭代不执行）

- 新建 Java 微服务仓、双栈并行上线、网关切流。
- Java 生产发布链路与跨服务事务治理。

## 影响

- 所有需求、缺陷、测试计划以 Node 执行线验收。
- Java 文档继续维护，但标记为“规划态”，不得作为当前开发任务来源。
- 文档索引默认入口改为本 ADR + `architecture-executable-v2.md`。

## 里程碑重申

1. Phase A：安全与租户隔离强约束收敛。  
2. Phase B：事件-指标-标签链路稳定化。  
3. Phase C：模块边界重构与可观测性补齐。  

## 退出条件（进入 Java 实施前必须满足）

- Node 执行线连续 2 个迭代回归稳定（无 P0/P1 架构回归）。
- 契约与数据模型冻结版本明确，并有迁移映射清单。
- Java skeleton + CI + 最小可运行服务在仓库内可验证。
