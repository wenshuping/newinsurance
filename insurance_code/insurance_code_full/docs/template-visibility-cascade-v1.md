# 模板可见性层级规则（v1）

更新时间：2026-03-06  
状态：`ACTIVE`

## 1. 规则结论

硬口径：

> 只有源模板状态为 `有效 / 进行中 / 已发布` 时，才允许向下一级下发；但“平台 -> 租户”和“租户 -> 员工”不是同一条规则。平台模板给租户看时仍按 `失效` 继承态展示；租户在本租户内发布后的模板，员工侧按真实状态展示。

1. 只有当“上一级”模板本身处于 `进行中 / 生效 / 已发布` 时，才允许向“下一级”下发。
2. 平台模板下发给租户账号时，状态必须按 `失效` 展示。
3. 租户账号在本租户内发布/进行中的模板，下发给员工时保持真实状态，不再强制改成 `失效`。
4. 客户端只看最终可投放内容，不看上级继承态。

## 2. 层级定义

1. 平台管理员
2. 租户账号（公司管理员）
3. 员工账号（团队主管 / 业务员）
4. 客户

## 3. 逐层规则

### 3.1 平台管理员 -> 租户账号

1. 平台创建的模板，只有源模板处于可见状态时，租户账号才可以看到。
2. 但租户账号看到时，状态必须展示为 `失效`。
3. 因为租户侧看到的是 `失效`，所以员工账号不能继续看到这条平台模板。

### 3.2 租户账号 -> 员工账号

1. 租户账号创建或继承后发布的模板，只要当前租户状态处于 `进行中 / 生效 / 已发布`，员工账号就可以看到。
2. 员工账号看到时保留真实状态，不再强制展示为 `失效`。
3. 平台模板如果还没有在当前租户形成 override，员工账号仍然不能直接看到。

### 3.3 员工账号 -> 客户

1. 客户端只看最终可见模板。
2. 对客户来说，只有模板状态是 `进行中 / 生效 / 已发布` 时才展示。
3. 继承态模板仍不直接展示给客户。
4. `失效 / 下线 / 结束` 一律不展示给客户。

## 4. 示例

### 示例 A

1. 平台管理员创建活动模板 `A`
2. 租户账号在 P 端可以看到 `A`
3. 租户账号看到的状态是 `失效`
4. 员工账号看不到 `A`
5. 客户端也看不到 `A`

### 示例 B

1. 租户账号创建学习模板 `B`
2. 员工账号在 B/C 端可以看到 `B`
3. 员工账号看到的状态仍然是 `发布 / 进行中`
4. 是否继续给客户展示，取决于终端投放口径

### 示例 C

1. 员工账号创建活动/内容/商城模板 `C`
2. 如果 `C` 的状态是 `进行中 / 生效 / 已发布`
3. 则客户端可以看到 `C`
4. 如果 `C` 是 `失效 / 下线 / 结束`
5. 则客户端不能看到 `C`

## 5. 实现口径

1. 可见性判断和层级关系，统一参考：`server/skeleton-c-v1/common/template-visibility.mjs`
2. 面向客户的最终状态可见性，统一参考：`server/skeleton-c-v1/common/status-policy.mjs`
3. 任何列表接口都不能自己重新发明一套“平台/租户/员工/客户”的过滤规则。

## 6. 接口映射

1. P 端模板管理视图
   1. `GET /api/p/activities`
   2. `GET /api/p/learning/courses`
   3. `GET /api/p/mall/products`
   4. `GET /api/p/mall/activities`
2. B 端模板管理视图
   1. `GET /api/b/activity-configs`
   2. `GET /api/b/content/items`
   3. `GET /api/b/mall/products`
   4. `GET /api/b/mall/activities`
3. C 端最终投放视图
   1. `GET /api/activities`
   2. `GET /api/learning/courses`
   3. `GET /api/mall/items`
   4. `GET /api/mall/activities`
4. C 端动作接口也必须复用同一口径
   1. `POST /api/activities/:id/complete`
   2. `POST /api/learning/courses/:id/complete`
   3. `POST /api/mall/redeem`
   4. `POST /api/mall/activities/:id/join`

## 7. 状态映射约定

1. 平台模板给租户账号时：活动继承态 `offline`
2. 平台模板给租户账号时：学习/商城继承态 `inactive`
3. 租户内部发布后的模板给员工时：返回真实状态
4. C 端不展示继承态，不返回给客户

## 8. 回归脚本

1. 冒烟脚本：`npm run test:smoke:template-visibility-cascade`
2. 断言内容：
   1. 平台模板对租户可见，但状态失效
   2. 平台模板不透传给员工
   3. 租户模板对员工可见，且保留发布/进行中状态
   4. 继承态模板不透传给客户

## 9. 开发约束

1. 如果是“上级模板给下级看”，要做的是：
   1. 先判断源模板状态是否属于 `进行中 / 生效 / 已发布`
   2. 只有满足条件才允许下级看到
   3. 下级看到后把展示状态映射成 `失效`
2. 如果是“给更下一级最终投放”，要做的是：
   1. 只认最终可见状态
   2. 不认继承态 `失效`
3. 新增接口时，先问自己：
   1. 这是“模板管理视图”
   2. 还是“最终投放视图”
4. 两种视图的过滤规则不同，不能混用。
