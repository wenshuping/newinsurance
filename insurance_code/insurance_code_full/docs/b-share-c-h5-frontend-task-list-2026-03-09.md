# B 端分享 C 端 H5 落地方案 前端任务单（2026-03-09）

## 1. 文档定位

本文档是 `/Users/wenshuping/Documents/New project/insurance_code/insurance_code_full/docs/b-share-c-h5-landing-solution-2026-03-09.md` 的前端拆解版。

目标：

1. 把分享能力拆成可执行的前端任务
2. 统一 H5 落地页、CTA、回跳、失效页的交互规则
3. 保证 B 端、分享 H5、C 端目标页之间的体验一致

## 2. 前端范围

本次前端范围只包含：

1. 统一分享落地页
2. 不同内容类型的分享模板
3. CTA 跳转逻辑
4. 登录回跳逻辑
5. 失效与兜底页面
6. B 端分享入口与弹窗

本次不做：

1. 分享海报生成
2. 个人中心、订单、核销类分享页
3. 在分享页直接执行兑换、核销、任务完成
4. 自定义多主题分享模板系统

## 3. 前端要支持的分享类型

第一阶段必须支持：

1. `activity`
2. `learning_course`
3. `mall_home`
4. `mall_item`

第二阶段再支持：

1. `learning_game`
2. `learning_tool`

## 4. 页面任务

## 4.1 统一分享落地页

建议新增统一路由：

- `/share/:shareCode`

任务：

1. 读取 `shareCode`
2. 调 `GET /api/share/:shareCode`
3. 根据 `shareType` 渲染对应内容
4. 根据 `valid` 渲染正常态或失效态

验收：

1. 任何分享类型都通过同一入口访问
2. 不允许为每种类型单独起一套分享页路由

## 4.2 正常态页面

页面要素：

1. 顶部品牌或来源标识
2. 封面图
3. 标签
4. 标题
5. 副标题
6. 亮点信息
7. CTA 主按钮
8. 次按钮或兜底按钮
9. 顾问来源说明

验收：

1. 页面风格接近 C 端
2. 内容缺失时有默认占位
3. 首屏能清楚表达“这是什么、下一步能做什么”

## 4.3 失效态页面

场景：

1. `shareCode` 不存在
2. `shareCode` 过期
3. `shareCode` 被禁用
4. 目标资源已下线

页面要素：

1. 失效提示
2. 返回首页按钮
3. 返回频道页按钮

验收：

1. 失效页不白屏
2. 用户始终有下一步可点

## 5. 组件拆分任务

建议组件：

1. `ShareLandingPage`
2. `ShareRenderer`
3. `ShareHero`
4. `ShareMeta`
5. `ShareCTA`
6. `ShareInvalidState`
7. `ShareExpiredState`

职责建议：

### `ShareLandingPage`

1. 拉分享详情
2. 管理 loading / error / invalid 状态
3. 决定是否显示 CTA

### `ShareRenderer`

1. 根据 `shareType` 选模板
2. 统一把数据转为页面需要的展示结构

### `ShareCTA`

1. 处理点击
2. 触发 click 埋点
3. 生成目标跳转地址

## 6. 跳转与回跳任务

## 6.1 跳转规则

前端必须遵循后端返回：

1. `targetCPath`
2. `fallbackCPath`
3. `loginRequired`

规则：

1. 有效且无需登录：直接跳 `targetCPath`
2. 有效且需要登录：先去登录页，再带 `redirect`
3. 目标页失效：跳 `fallbackCPath`
4. 没有 fallback：最后跳首页 `/`

## 6.2 登录回跳

任务：

1. 统一 `redirect` 参数格式
2. 登录成功后跳回目标页
3. 登录失败时保留在登录页或回分享页

验收：

1. 活动、课程、商城三类分享都走同一套回跳逻辑
2. 不同前端页面不各自实现一套回跳规则

## 6.3 目标页映射

前端需要配合的目标页如下：

1. `activity`
   - `/activities?activityId=:id`
2. `learning_course`
   - `/learning?courseId=:id`
3. `learning_game`
   - `/learning?tab=games&gameId=:id`
4. `learning_tool`
   - `/learning?tab=tools&toolId=:id`
5. `mall_home`
   - `/mall`
6. `mall_item`
   - `/mall?itemId=:id`

说明：

1. 前端不应重新发明目标路径
2. 以后如果路径调整，应优先以后端返回值为准

## 7. B 端任务

## 7.1 分享入口

第一阶段接入位置：

1. 活动详情页
2. 课程详情页
3. 商城首页
4. 商品详情页

任务：

1. 增加“分享”按钮
2. 点击后调用 `POST /api/b/shares`
3. 展示分享结果

## 7.2 分享弹窗

弹窗至少展示：

1. 分享标题
2. 分享链接
3. 二维码占位
4. 复制按钮
5. 关闭按钮

第一阶段二维码可先不做生成，只保留位置和文案。

验收：

1. 业务员不需要手工拼链接
2. 复制按钮一键可用
3. 后端返回错误时有明确提示

## 8. H5 模板任务

建议模板映射：

1. `activity` -> 活动详情模板
2. `learning_course` -> 课程详情模板
3. `learning_game` -> 游戏卡片模板
4. `learning_tool` -> 工具卡片模板
5. `mall_home` -> 商城频道模板
6. `mall_item` -> 商品详情模板

原则：

1. 模板数量不要太多
2. 视觉要像 C 端，不要像后台页
3. 不要把弹层页面直接粗暴截图式搬到 H5
