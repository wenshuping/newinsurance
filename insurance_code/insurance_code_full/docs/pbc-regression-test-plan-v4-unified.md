## 2. 前序问题与修正说明（必须先读）

### 2.1 已识别问题

1. 端口冲突：历史文档中 P/C 端口互换，执行人按不同文档会跑错端口。  
2. 指标口径冲突：“30天登录次数”与“30天登录天数”混用，验收标准不唯一。  
3. 模板可见规则冲突：一处写“按 company_admin 上架可见”，一处写“按归属业务员可见”。  
4. 测试账号策略冲突：固定账号、动态账号、脚本内账号并存，复现不稳定。  
5. 权限矩阵漏测：缺少“公司 admin 页面权限勾选/取消后菜单与接口生效”验证。

### 2.2 本版修正

1. 固定端口基线：`API=4000 / C=3000 / B=3002 / P=3015`。  
2. 指标口径按当前需求收敛：重点验证 `30天签到天数` 与连续签到/登录天数。  
3. 模板可见按当前实现统一：客户仅看“归属业务员创建且生效”的模板。  
4. 账号采用“动态创建 + 最小固定种子账号（platform001）”双模式。  
5. 新增“页面权限矩阵”完整回归与 API 校验。

## 4. 数据前置规范（统一）

每轮执行创建一套唯一数据（`<ts>` 时间戳后缀）：

1. 租户：`REG_TENANT_<ts>`  
2. 公司管理员：`tenant_<ts>@demo.local`  
3. 业务员A：`agent_a_<ts>@demo.local`  
4. 业务员B：`agent_b_<ts>@demo.local`  
5. 客户A：`139<8位>`  
6. 客户B：`138<8位>`

说明：
- 禁止复用上轮租户与客户作为唯一验证样本。  
- 禁止离线改内存态替代在线 API 结果判定（当前后端基线为 `STORAGE_BACKEND=postgres`）。

## 5. 权限与可见规则（统一口径）

### 5.1 客户归属规则（当前版本）

1. 一个客户只归属一个业务员（`ownerUserId` / `owner_agent_id`）。  
2. 业务员仅可见归属给自己的客户。  
3. 团队主管可见本团队客户。  
4. 公司管理员可见本租户全部客户。

### 5.2 模板可见规则（当前实现）

1. 客户端：仅可见“当前客户归属业务员创建且生效”的模板。  
2. 业务员/团队主管：可见本租户生效模板以及自己创建的模板
3. 公司管理员：可见平台模板 + 本租户模板。  
4. 平台管理员：仅可见自己创建的平台模板。
5. 平台创建生效模板公司admin自动可以看到 但是模板状态是失效的 且有平台模板标志
### 5.3 页面权限矩阵（公司 admin）

1. 左侧公司列表 = 租户列表。  
2. 权限配置矩阵作用对象 = `company_admin` 页面访问权限。  
3. 勾选页面后：菜单可见且可访问对应页面/接口。  
4. 取消勾选后：菜单隐藏或进入被拦截（前后端都要生效）。

## 7. 关键 API 与数据核验点

### 7.1 API 核验

```bash
# agent 客户列表范围
curl -s 'http://127.0.0.1:4000/api/b/customers' \
  -H 'x-actor-type: agent' -H 'x-actor-id: <agentId>' -H 'x-tenant-id: <tenantId>'

# 客户按手机号分配
curl -s -X POST 'http://127.0.0.1:4000/api/p/customers/assign-by-mobile' \
  -H 'Content-Type: application/json' \
  -H 'x-actor-type: employee' -H 'x-actor-id: <companyAdminId>' -H 'x-tenant-id: <tenantId>' \
  --data '{"mobile":"<mobile>","agentId":<agentId>}'

# 页面权限矩阵读取/保存
curl -s 'http://127.0.0.1:4000/api/p/permissions/company-admin-pages?tenantId=<tenantId>' \
  -H 'x-actor-type: employee' -H 'x-actor-id: <platformAdminId>' -H 'x-tenant-id: 1'
```

### 7.2 数据核验（SQL）

```sql
-- 客户归属核验
select id, tenant_id, owner_agent_id, updated_at
from c_customers
where tenant_id = <tenant_id>
order by id desc
limit 50;

-- 分享分端日统计
select stat_date, metric_key, actor_id, cnt
from p_metric_counter_daily
where tenant_id = <tenant_id>
  and stat_date = '<yyyy-mm-dd>'
  and metric_key in ('c_share_success_cnt', 'b_share_success_cnt');
```

## 8. 出口标准（Go/No-Go）

必须全部满足：
1. 全部 `P0` 用例通过率 `100%`。  
2. `P1` 用例通过率 `>= 95%`，且无数据权限类缺陷。  
3. 无跨租户数据泄露。  
4. 无积分账不平、重复扣分/重复发分。  
5. 指标口径不出现“次数/天数混算”。
