## 2. Top 5 优先风险与动作

### R6 进度失控（最高）
- 本周动作：冻结需求，禁止新增非C端P0功能。
- 验收信号：每日燃尽按计划下降。

### R1/R2 性能与一致性
- 本周动作：完成签到/兑换/核销压测与冲突测试。
- 验收信号：P95 <= 800ms，幂等冲突全部被拦截。

### R5 数据库稳定性
- 本周动作：慢SQLTOP10治理，连接池告警上线。
- 验收信号：连接池占用<80%，无长事务。

### R10 迁移风险
- 本周动作：staging完整迁移演练2次。
- 验收信号：关键表计数与余额核对100%通过。

## 3. 风险例会机制

- 频率：每日站会 + 每周风险复盘。
- 输入：告警、缺陷、联调阻塞、变更申请。
- 输出：风险状态（Open/Mitigated/Closed）与责任人截止日期。

## 4. 风险治理决策与取舍

- 决策：优先保C端P0上线，B/P需求严格控范围。
- 取舍：短期功能完整性下降，换取主流程上线确定性。
- 决策：高风险链路（积分/兑换/核销）优先自动化测试。
- 取舍：前期测试建设投入增加，但显著降低生产事故概率。

## 5. 落地实施步骤与负责人建议

1. 项目经理每日更新风险台账并同步责任人。
2. 后端负责人对R1/R2/R5设置技术里程碑与验收指标。
3. DevOps负责人对R4/R8执行一次故障演练。
4. QA负责人对R7/R10执行联调与迁移专项回归。
5. 架构师每周主持风险评审，关闭或升级风险项。
