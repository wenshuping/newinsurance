# P/B/C 端端到端回归测试文档（从创建租户开始）

文档版本：v1.0  
适用项目：`.` + `insurance_code_P` + `insurance_code_B`  
测试目标：验证“创建租户账号 -> P端配置 -> B端运营 -> C端可见与可用 -> 事件/指标/标签联动”完整闭环稳定。

## 1. 测试范围

- P端：租户管理、员工管理、内容与营销、事件管理、指标配置、标签列表、标签规则库。
- B端：员工登录、客户库、客户详情、内容管理、活动配置、积分商城、客户轨迹展示。
- C端：实名认证、学习、活动、签到、积分商城、兑换、个人中心。
- 联动：
  - P端/B端创建内容是否在C端可见。
  - C端行为埋点是否在B端客户行为轨迹/互动轨迹可见。
  - 标签规则（固定值/映射值）执行是否产生日志与输出值。

## 2. 环境与启动

## 2.1 服务地址建议

- API：`http://127.0.0.1:4000`
- P端：`http://127.0.0.1:3000`
- B端：`http://127.0.0.1:3001`
- C端：`http://127.0.0.1:3002`

## 2.2 启动命令

1. 启动 API（必须先启动）
```bash
cd .
node server/skeleton-c-v1.mjs
```

2. 启动 P 端
```bash
cd "$P_REPO_PATH"
npm run dev -- --port 3000 --host 0.0.0.0
```

3. 启动 B 端
```bash
cd "$B_REPO_PATH"
npm run dev -- --port 3001 --host 0.0.0.0
```

4. 启动 C 端
```bash
cd .
npm run dev -- --port 3002 --host 0.0.0.0
```

## 2.3 测试账号

- P端登录：
  - `platform001 / 123456`
  - `company001 / 123456`
  - `agent001 / 123456`
- B端登录：
  - `company001 / 123456`
  - `agent001 / 123456`
- C端登录：手机号 + 验证码流程（开发环境验证码默认 `123456`）。

## 3. 回归执行顺序（必须按顺序）

1. 创建新租户账号（P端）。
2. 在P端创建该租户员工账号（管理员/业务员）。
3. 用该员工账号登录B端，配置内容/活动/积分商品。
4. 用C端客户账号进入并验证内容可见、可参与、可兑换。
5. 回到B端客户详情，验证行为轨迹/互动轨迹/积分流水。
6. 回到P端验证事件定义、指标、标签规则执行与日志。

## 6. 验收通过标准

- P0 用例通过率 `100%`。
- P1 用例通过率 `>= 95%`，且无阻断缺陷。
- 不允许出现以下阻断问题：
  - 数据不同步（B端发布内容C端不可见）。
  - 埋点事件缺失（C端关键行为无法在B端/P端观察）。
  - 规则执行无日志或输出值错误。
  - 租户越权读取/修改。
